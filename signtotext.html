Sign to Text Converter


This project aims to develop a sign language to text converter using the combination of Mediapipe and LSTM (Long Short-Term Memory) algorithms. The goal is to create a system that can accurately recognize and interpret sign language gestures captured through a video feed and convert them into corresponding text output. This technology has the potential to bridge communication gaps between individuals who are deaf or hard of hearing and those who do not understand sign language.

Objective:

Develop a robust hand gesture recognition system using Mediapipe, an open-source framework for building real-time perception pipelines.

Implement an LSTM-based model to capture the temporal dependencies of sign language gestures and convert them into textual representation.

Create a user-friendly interface for capturing and processing video input in real-time.

Achieve high accuracy and real-time performance to ensure effective communication between sign language users and non-sign language users.

Methodology Implemented

Dataset collection: Gather a diverse dataset of sign language gestures, recorded in various lighting conditions and performed by different individuals.

Preprocessing: Normalize and preprocess the dataset, including resizing the hand region and extracting key hand landmarks using the Mediapipe framework.

Training LSTM model: Train an LSTM-based model using the preprocessed dataset, capturing the sequential information of hand gestures.

Integration: Integrate the trained LSTM model with the hand gesture recognition system built using Mediapipe.

Real-time processing: Develop an efficient real-time video processing pipeline to detect, recognize, and convert sign language gestures into text using the integrated system.

Evaluation: Assess the performance of the sign language to text converter by measuring accuracy, recognition speed, and robustness to variations in lighting, hand orientation, and user-specific gestures.

Technology Stack

Python: The programming language used for implementing the machine learning models, processing the video feed, and handling the backend logic.

OpenCV: An open-source computer vision library used for capturing and processing the video feed. OpenCV provides functionalities for image and video manipulation, including tasks like image preprocessing, feature extraction, and gesture recognition.

Mediapipe: A cross-platform framework developed by Google that offers a wide range of pre-built components for building multimodal ML pipelines. In this case, Mediapipe can be utilized for hand tracking and pose estimation from the video feed. It provides ready-to-use models and APIs for real-time hand detection and tracking, which are crucial for recognizing sign language gestures.

Long Short-Term Memory (LSTM): A type of recurrent neural network (RNN) that can capture long-term dependencies in sequential data. LSTM networks are well-suited for tasks involving sequential or time-series data, making them a suitable choice for recognizing sign language gestures. Python libraries like TensorFlow or PyTorch can be used to implement the LSTM model.

Deep Learning Framework: Depending on the choice of LSTM implementation, you can utilize popular deep learning frameworks such as TensorFlow or PyTorch. These frameworks provide a rich set of tools and APIs for building, training, and evaluating deep learning models.
