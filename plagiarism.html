Plagiarism Checker


The approach preprocesses source codes by removing punctuation, tokenizing, removing stopwords, and stemming. TF-IDF vectors capture term importance. Similarity is determined by cosine similarity between a user's code and existing codes. Codes above a threshold (0.8) are considered similar. This enables code reuse, plagiarism detection, and knowledge sharing. Effectiveness relies on preprocessing quality, TF-IDF representation, and adjustable thresholds.

Objective:

The objective of the implemented methodology is to preprocess and analyze code data to identify similar codes based on their semantic content.

This involves transforming codes into TF-IDF vectors, calculating cosine similarity, and identifying codes with high similarity to a user-entered code.

The goal is to support code reuse, plagiarism detection, and knowledge sharing among developers.

Methodology Implemented

Reading the codes from a CSV file and storing them in a list (codes).

Preprocessing the codes by removing punctuation, tokenizing, removing stopwords, and stemming the tokens.

Creating TF-IDF vectors for the preprocessed codes using TfidfVectorizer from scikit-learn.

Accepting user input for a code. Preprocessing the user-entered code using the same preprocessing steps as the codes.

Calculating the TF-IDF vector for the preprocessed user code. Calculating the cosine similarity between the user code's TF-IDF vector and each of the preprocessed codes' TF-IDF vectors.

Printing the codes that have a cosine similarity greater than 0.8, indicating high similarity to the user-entered code.

Technology Stack

Python:
The programming language used for implementing the machine learning models, processing the video feed, and handling the backend logic.

Natural Language Processing (NLP) Libraries:
NLTK (Natural Language Toolkit): A popular library for NLP tasks in Python. NLTK provides a wide range of functionalities for tokenization, stemming, and stopword removal, which are essential for preprocessing the source code. scikit-learn: A machine learning library that includes various tools for text preprocessing and feature extraction, such as TF-IDF vectorization.

TF-IDF (Term Frequency-Inverse Document Frequency):
A numerical statistic that reflects the importance of a term in a collection of documents. Libraries such as scikit-learn provide built-in methods for calculating TF-IDF scores, which can be used to represent the source code as a vector.

Cosine Similarity:
A metric used to determine the similarity between two vectors. The cosine similarity is commonly used in text similarity tasks. It can be calculated using libraries like scikit-learn or manually implemented using NumPy.

Flask/Django:
A web framework in Python that can be used to develop the backend server for handling client requests and serving the sign language to text conversion functionality. Flask or Django can provide the necessary endpoints to receive video input, process it using the Mediapipe and LSTM algorithms, and return the corresponding text output.
